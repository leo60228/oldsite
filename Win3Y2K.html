Windows 3.x Y2K

In Windows 3.x (here-forth referred to as Win3), 2000 would display as 19:0. The reason for this is because of how monitors work. The original protocol for displaying data on monitors is called VGA. The mode of VGA Win3 (as well as DOS) uses is called 80x25 Text Mode. The screen is a rectangle divided into cells, 80 cells wide and 25 cells tall. Every cell holds one character (one letter/symbol/number), and uses two bytes (a number from 0 to 255) of memory. One to designate which character it is, and one to designate what color it is. The character is stored in a protocol called ASCII. In ASCII, every character is a number, from 0 to 255. Every number character is next to each other. Let's say 0 is at 10, 1 at 11, two at 12, etc. Win3 adds the desired digit to 10. For colors, there are sixteen colors, no special protocol. Back to ASCII, let's say 20 is colon. Win3 will try to display 19100 (because it can't handle millennia), but because years being 4 digits is hardcoded (can't be changed) it will try to display 10 as one digit, by adding 10 to 10. As said before, twenty is a colon, so it is displayed as 19:0.
